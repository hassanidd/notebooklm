Product concept

This app allows a user to:

Upload a document or any supported file type

Automatically detect file type

Extract content from the file

Normalize and clean the extracted content

Chunk the content

Build “embed text” for each chunk

Build metadata for each chunk

Generate embeddings

Index into a vector database

Make the content searchable for retrieval

Optionally use the processed content later in a RAG system or LLM training pipeline

The app has two ingestion modes:

Auto Mode
The system runs the entire ingestion pipeline automatically with minimal user interaction.

Guided Mode
The system pauses at key stages and lets the user review, edit, approve, and configure outputs before moving to the next stage.

Important product behaviors to reflect in UI

The designs must communicate the following product rules clearly:

Auto Mode behavior

fast, minimal friction

system automatically processes all steps

user mostly monitors progress

user can inspect outputs but not necessarily edit at every stage

Guided Mode behavior

system pauses at key checkpoints

user can review, edit, and approve before continuing

much more control over extraction, chunks, embedding strategy, metadata, and permissions

Build embed text logic

The UI should reflect that embed text is not always the same as raw text:

plain text chunks can use cleaned raw text

table chunks may use normalized textual representation

image chunks may use OCR + caption

OCR-heavy chunks may use constrained summaries

mixed chunks may use composed embed text
